# StayThread Beta Usability Loop

This document is the operating artifact for the first site-owner beta. Use it with `docs/beta-test-plan.md`.

## Objective

Validate whether StayThread helps beginner/intermediate solo international site owners keep moving on slow-feedback SEO/Growth work without requiring them to store private keywords, backlink URLs, domains, or outreach targets.

The product category remains broader than SEO. This loop validates the first wedge.

## Day 0: Assessment And Setup

Goal: prove the public value path works before account creation.

Required observations:

- User understands StayThread as a long-term action system, not only an SEO tool.
- User understands the current beta focus is site-owner SEO/Growth.
- User completes Assessment without login.
- Result stage/depth/bottlenecks feel plausible enough to continue.
- User can select or describe current site stage.
- User can identify whether the first workflow should start with keyword discovery, backlink work, or content pipeline.
- User understands real keywords and backlink URLs can stay outside StayThread.
- User decides whether to register/save after seeing the prescription.

| Assessment completion time |  |

| Registered after result | yes/no |

Pass evidence:

- User can explain what their first 7-day action line is.
- User does not ask why the product needs real keywords or real URLs.
- User sees registration as a save step, not a forced signup wall.

## Day 7: Weekly Review And Interview

Goal: validate realism, privacy, and recovery.

Quantitative check:

| Signal | Value |
| --- | --- |
| Completed days in first 7 |  |
| Completed at least 5 of 7 | yes/no |
| Used keep-alive at least once | yes/no |
| Recovered within 48h after miss | yes/no/not applicable |
| At least one process asset accumulated | yes/no |
| Weekly review completed | yes/no |

Interview questions:

- Did the tasks feel like real site-owner work or generic productivity advice?
- Which task felt most useful: keyword analysis, backlink screening, content pipeline, or review?
- Did count-only logging protect your private work product?
- Did any task ask for data you did not want to share?
- Was the keep-alive task small enough to restart after interruption?
- Did the weekly review say anything useful?
- What tutorial would have helped this week?

Pass evidence:

- User says at least one task resembles a realistic daily site-owner action.
- User can name a concrete process asset accumulated during the week.
- User accepts the privacy boundary.

## Day 14: Exit Interview

Goal: decide whether to continue, change, or stop the wedge.

Quantitative check:

| Signal | Value |
| --- | --- |
| Completed days in 14 |  |
| Completed 5 of first 7 | yes/no |
| Completed at least 9 of 14 | yes/no |
| Day-14 still active | yes/no |
| Process assets accumulated |  |
| Willing to continue | yes/no/maybe |
| Willing to pay for continuity | yes/no/maybe/price |
| Interested in keyword tutorial membership | yes/no/maybe |
| Interested in backlink tutorial membership | yes/no/maybe |

Exit interview questions:

- What did StayThread help you do that you were not doing before?
- Which part should be removed?
- Which task repeated too much?
- Did the product avoid promising rankings/traffic/income?
- Would you continue if there were no integrations?
- Would you pay for the continuity system alone?
- Would you pay for Semrush/Ahrefs/Trends keyword discovery tutorials?
- Would you pay for backlink screening/outreach tutorials?
- What price would feel fair?

Continue criteria:

- User completed the first assessment.
- User completed Day-1 task.
- User completed at least 5 of first 7 days or gave strong evidence that keep-alive recovery worked.
- User accumulated at least one process asset.
- User says tasks are site-owner realistic, not generic productivity advice.
- Privacy boundary is accepted.
- At least some users show willingness to continue or pay.

Stop or revise criteria:

- Users see tasks as generic advice.
- Users refuse count-only logging or find it pointless.
- Users want actual keyword/backlink targets more than continuity support.
- Users do not return after missed days.
- Users cannot connect tasks to real SEO/Growth work.
