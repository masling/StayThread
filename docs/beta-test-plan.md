# StayThread Private Beta 测试计划

## 目标

验证 StayThread 是否能在不依赖压力、羞耻或复杂 productivity dashboard 的情况下，帮助初级到中级出海个人独立站长持续推进 SEO/Growth slow-feedback work，尤其是通过 Semrush、Ahrefs、Google Trends 等工具做关键词分析/发现，以及持续推进 SEO 外链工作。

## Cohort

- 5 到 10 名用户。
- 14-day beta。
- 6 到 8 名初级/中级出海个人独立站长：英文独立站、内容站、Affiliate/SEO 小站、niche site 或个人 SaaS 站运营者。
- 1 到 2 名 SEO/Growth 相邻用户可作为对照，例如正在做英文内容增长的 independent builder 或 writer。
- 个人 SaaS 站长可纳入；成熟增长团队或已有专职 SEO 团队的站点暂不作为首批目标。
- 不把 broad personal growth users 作为首批 Beta 验证对象。

## Entry Criteria

每个参与者需要有一个自己真实在意的出海独立站或 SEO/Growth 项目，并愿意连续 14 天记录 daily state、SEO/Growth action、工作量数据和 completion level。系统默认不要求用户提交真实关键词或真实外链 URL。

## 招募筛选问题

- 你现在运营的网站类型是什么：英文独立站、内容站、Affiliate site、SaaS landing/site、还是其他？
- 你的网站阶段是什么：刚建站、已有少量内容、已有自然流量、正在增长、还是遇到增长瓶颈？
- 你目前最卡的是关键词研究、内容生产、外链推广、技术 SEO、还是持续执行？
- 你是否是个人独立推进 SEO/Growth，还是已经有成熟团队/专职 SEO 负责？
- 你平时是否使用或愿意打开 Semrush、Ahrefs、Google Trends 等工具做关键词分析？
- 你每周能投入多少时间在 SEO/Growth 上？
- 过去 30 天你是否统计过自己分析了几个网站、几个词根、多少关键词、几个外链机会？
- 你是否愿意连续 14 天每天完成一个 standard、easy 或 keep-alive SEO/Growth action？
- 你是否接受平台只记录工作量和可选编号/元数据，不要求保存真实关键词或真实外链地址？

## 测试脚本

Day 0：

- 从未登录 Landing 进入。
- 点击 `Start free assessment`。
- 完成 Assessment。
- 录入网站阶段、当前 SEO bottleneck、可投入时间和已有 process assets。
- 选择当前优先路径：刚开始站点优先关键词发现；已建站但内容少/流量少优先加入 SEO 外链工作。
- 查看 stage、depth、bottlenecks 和 SEO/Growth prescription。
- 判断是否愿意注册保存结果。
- 注册后进入 Today。

Days 1-7：

- 每天打开 Today。
- 选择 current state。
- 完成 standard、easy 或 keep-alive SEO/Growth task。
- 关键词任务围绕打开 Semrush/Ahrefs/Trends、分析竞品/网站、分析词根、分析关键词、判断可用关键词数量。
- 外链任务围绕筛选正常网站、判断是否有申请通道、判断是否可能通过、排除垃圾外链站、记录 outreach/follow-up 工作量。
- 记录 completion level。
- 记录至少一种 work evidence：tools opened、competitor/site count analyzed、seed terms analyzed、keywords analyzed、usable keyword count found、backlink channels screened、normal-site prospects counted、outreach attempts or follow-ups logged。
- 真实关键词和真实外链地址不强制入库；如需记录，默认使用编号和可选元数据。
- 回答 daily review prompts。

Day 7：

- 完成 weekly review。
- 访谈：任务是否像站长每天真的可以做的小动作，而不是 generic productivity advice。
- 访谈：关键词分析流程是否可执行、外链筛选标准是否清楚、keep-alive task 是否能减少中断后的重启成本。
- 访谈：不保存真实关键词/外链 URL，只记录工作量和编号，是否符合站长对成果隐私的预期。

Days 8-14：

- 继续 daily loop。
- 观察 missed day 后是否能用 keep-alive task 在 48 小时内 recovery。
- 观察用户是否继续完成关键词分析、竞品/网站分析、词根分析、外链机会筛选或 outreach follow-up。

Day 14：

- 访谈 clarity、trust、task realism、registration timing、SEO/Growth 专业度，以及产品是否减少了 restart behavior。
- 访谈用户是否愿意继续使用或为 continuity system 付费。
- 访谈用户是否愿意为 Semrush/Ahrefs/Trends 关键词发现教程、外链筛选教程等会员内容付费，并收集意见反馈。

## Quantitative Signals

- Landing-to-assessment start rate。
- Assessment completion。
- Result-to-register conversion。
- Day-1 completion。
- Completion level distribution。
- Five-of-seven chain rate。
- Missed-day recovery within 48 hours。
- Weekly review completion。
- SEO/Growth work evidence accumulated：
  - tools opened。
  - competitor/site count analyzed。
  - seed terms analyzed。
  - keywords analyzed。
  - usable keyword count found。
  - backlink channels screened。
  - normal-site prospects counted。
  - outreach attempts or follow-ups logged。

## Qualitative Questions

- Landing 是否让你理解 StayThread 不是普通 habit tracker？
- 先测试 Assessment、再注册保存的顺序是否自然？
- Assessment result 是否足够准确，能不能建立 trust？
- Recommended depth 是太容易、太难，还是刚好？
- Keep-alive task 是否减少 all-or-nothing thinking？
- SEO/Growth task 是否像是针对站长真实工作生成，而不是 generic productivity advice？
- 关键词任务是否促使你打开 Semrush/Ahrefs/Trends，并完成网站、词根、关键词分析？
- 关键词任务是否能帮助你找到若干可用关键词，即使真实关键词不记录在平台？
- 外链任务是否能帮助你筛掉垃圾外链站，并找到有申请通道、可能通过、正常内容网站的机会？
- 只记录编号、数量和可选元数据，不保存真实关键词/外链地址，是否让你更愿意使用？
- 这些任务是否避免了承诺排名、流量或收入结果？
- AI 只对工作数据做鼓励式复盘、不输出关键词和外链对象，这个边界是否合理？
- 你是否愿意为关键词发现和外链筛选教程付费？希望教程包含什么？
- Today page 早上打开时是否足够简单？
- 哪些 copy 感觉 supportive，哪些感觉多余或烦？

## Beta Exit Criteria

进入 production build 的建议门槛：

- 至少 70% 的用户完成 Assessment。
- 至少 50% 的用户在看到 SEO/Growth prescription 后愿意注册保存。
- 至少 60% 完成 Day-1 SEO task。
- 至少 50% 在 7 天内完成 5 天有效 SEO/Growth action。
- 每位用户 7 天内至少累计一种 work evidence：tools opened、competitor/site count analyzed、seed terms analyzed、keywords analyzed、usable keyword count found、backlink channels screened、normal-site prospects counted、outreach attempts or follow-ups logged。
- 用户能解释 standard、easy、keep-alive 三档为什么存在。
- 用户主观认为任务像站长每天真的可以做的小动作，而不是 generic productivity advice。
- 用户认为不保存真实关键词/外链地址的隐私边界合理。
- Day 14 至少收集到是否愿意为 continuity system 或 SEO 教程会员服务付费的明确反馈。
- 用户主观反馈 recovery 比过去使用的工具更容易。
