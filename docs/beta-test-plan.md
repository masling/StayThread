# StayThread Private Beta 测试计划

## 目标

验证 StayThread 是否能在不依赖压力、羞耻或复杂 productivity dashboard 的情况下，帮助用户维持 long-term action。

## Cohort

- 5 到 10 名用户。
- 14-day beta。
- 优先选择 independent builders、site owners、readers、writers 和 gentle movement restarters。

## Entry Criteria

每个参与者需要有一个自己真实在意的 slow-feedback goal，并愿意连续 14 天记录 daily state 和 completion level。

## 测试脚本

Day 0：

- 从未登录 Landing 进入。
- 点击 `Start free assessment`。
- 完成 Assessment。
- 查看 stage、depth、bottlenecks 和 prescription。
- 判断是否愿意注册保存结果。
- 注册后进入 Today。

Days 1-7：

- 每天打开 Today。
- 选择 current state。
- 完成 standard、easy 或 minimum task。
- 记录 completion level。
- 回答 daily review prompts。

Day 7：

- 完成 weekly review。
- 访谈：minimum task 是否显得有效、太小、还是有帮助。

Days 8-14：

- 继续 daily loop。
- 观察 missed day 后是否能在 48 小时内 recovery。

Day 14：

- 访谈 clarity、trust、task realism、registration timing，以及产品是否减少了 restart behavior。

## Quantitative Signals

- Landing-to-assessment start rate。
- Assessment completion。
- Result-to-register conversion。
- Day-1 completion。
- Completion level distribution。
- Five-of-seven chain rate。
- Missed-day recovery within 48 hours。
- Weekly review completion。
- Process assets accumulated。

## Qualitative Questions

- Landing 是否让你理解 StayThread 不是普通 habit tracker？
- 先测试 Assessment、再注册保存的顺序是否自然？
- Assessment result 是否足够准确，能不能建立 trust？
- Recommended depth 是太容易、太难，还是刚好？
- Minimum task 是否减少 all-or-nothing thinking？
- Category task 是否像是针对 goal type 生成，而不是 generic advice？
- Today page 早上打开时是否足够简单？
- 哪些 copy 感觉 supportive，哪些感觉多余或烦？

## Beta Exit Criteria

进入 production build 的建议门槛：

- 至少 70% 的用户完成 Assessment。
- 至少 50% 的用户在看到 result 后愿意注册保存。
- 至少 60% 完成 Day-1 task。
- 至少 50% 达成 five-of-seven chain。
- 用户能解释 standard、easy、minimum 三档为什么存在。
- 用户主观反馈 recovery 比过去使用的工具更容易。
